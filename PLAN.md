# Anti Planer — 작업 계획 및 체크리스트

이 파일은 여러 AI 모델(Claude, Gemini 등)이 협업하며 작업 진행 상황을 추적하기 위한 마스터 플랜입니다. 새로운 요청이 있을 때마다 여기에 할 일을 정리하고, 하나씩 해결하며 체크해 나갑니다.

---

## 🎯 핵심 목표
- **일관성 유지**: 여러 모델을 오가며 작업해도 맥락을 잃지 않도록 작업 상태 기록
- **체계적 진행**: 사용자 요청을 태스크 단위로 쪼개어 순차적으로 완수

## 💡 협업 프로세스 (AI 규칙)
1. **정리**: 사용자가 새로운 기능을 요청하면 먼저 `PLAN.md`의 `PHASE 2`에 세부 항목으로 추가합니다.
2. **확인**: 추가된 리스트를 사용자에게 보여주고 진행 순서를 확인받습니다.
3. **수행**: 한 번에 하나의 체크리스트 아이템만 집중해서 개발합니다.
4. **기록**: 작업을 시작할 때 시작 시각과 모델명을 기록하고, 종료 시 종료 시각과 결과를 반드시 남깁니다.
   - 형식: `> 🕐 시작: YYYY-MM-DD HH:MM · 모델명 → 🕑 종료: HH:MM | 결과`
   - 결과 표기: `✅ 성공` / `❌ 실패 (사유)` / `⚠️ 중단 (할당량 소진 등)`
   - 모델이 중간에 끊겨도 다음 모델이 로그를 보고 어디서 멈췄는지 파악할 수 있습니다.
5. **업데이트**: 작업이 완료되면 해당 항목을 `[x]` 처리하고, 필요시 `최근 업데이트` 섹션으로 이동시킵니다. 로그는 그대로 유지합니다.
6. **동기화**: 작업 결과가 `README.md`의 규칙에 어긋나지는 않았는지 최종 점검합니다.
7. **확인2**: `PLAN.md` 파일에 작업 확인 체크하고 마무리한다.
