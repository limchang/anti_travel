# Anti Planer — 작업 계획 및 체크리스트

이 파일은 여러 AI 모델(Claude, Gemini 등)이 협업하며 작업 진행 상황을 추적하기 위한 마스터 플랜입니다.

## 🎯 핵심 목표
- **일관성 유지**: 여러 모델을 오가며 작업해도 맥락을 잃지 않도록 작업 상태 기록
- **체계적 진행**: 사용자 요청을 태스크 단위로 쪼개어 순차적으로 완수
- **UI/UX 정밀도**: README.md의 규칙을 준수하며 정교한 UI 구현

## � 협업 프로세스 (AI 규칙)
1. **정리**: 사용자가 새로운 작업을 요청하면 `현재 할 일 목록` 맨 위에 새 번호로 추가합니다.
2. **확인**: 추가된 항목을 사용자에게 보여주고 진행 여부를 확인받습니다.
3. **수행**: 한 번에 하나의 체크리스트 아이템만 집중해서 개발합니다.
4. **기록**: 작업 시작 시 시작 시각·모델명을, 종료 시 종료 시각·결과를 반드시 남깁니다.
   - 형식: `> 🕐 시작: YYYY-MM-DD HH:MM · 모델명 → 🕑 종료: HH:MM | 결과`
   - 결과 표기: `✅ 성공` / `❌ 실패 (사유)` / `⚠️ 중단 (할당량 소진 등)`
   - 모델이 중간에 끊겨도 다음 모델이 로그를 보고 어디서 멈췄는지 파악할 수 있습니다.
5. **업데이트**: 완료된 항목은 `[x]` 처리합니다. 로그는 그대로 유지합니다.
6. **동기화**: 작업 결과가 `README.md`의 규칙에 어긋나지 않는지 최종 점검합니다.
7. **새 작업은 목록 맨 위에 추가**: 최신 작업이 항상 상단에 위치하도록 합니다.
8. **사용자 후속 요청 규칙**: 사용자가 별도 문제 제기 없이 다음 작업을 이어서 요청하면, 직전의 확인 대기/개선 중 항목은 사용자 확인이 끝난 것으로 보고 완료 처리한 뒤 다음 지시를 진행합니다.
9. **요청 분석 우선 기록**: 사용자의 질문/요청을 받으면 작업 전에 먼저 그 요청을 어떻게 이해했는지 `PLAN.md`에 기록합니다.
10. **작업 계획 기록 후 작업 시작**: 요청 분석 기록 다음에는 실행 계획을 한 번 더 구체화해 `PLAN.md`에 추가 기록하고, 그다음에 실제 수정/작업을 시작합니다.
11. **압축/요약 요청 처리 규칙**: 사용자가 특정 작업 번호 구간을 압축하거나 요약하라고 하면, 별도 요약 섹션을 새로 만들지 말고 해당 기존 항목 자리 자체를 한 줄 요약 형태로 줄여 정리합니다.
12. **정렬은 생명**: 버튼 끝선, 카드 폭, 행 정렬, 여백 기준선처럼 눈에 보이는 정렬은 최우선으로 맞춥니다. 이 기준을 놓치면 같은 일을 반복하게 되므로, UI 수정 시 기능과 함께 정렬 상태를 반드시 같이 점검합니다.
