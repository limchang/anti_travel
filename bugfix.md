# Bug Fix Log

## 2026-03-13

### Bug
- 숙박 다음날 첫 일정에서 운영 경고를 눌러 시작 시간을 자동 보정해도, 남는 시간이 이동칩 보정시간으로 반영되지 않고 `0분`으로 남는 경우가 있었음.

### Cause
- 보정시간 계산과 고정 시작시간 계산이 `07:33` 같은 시계 시각만 보고 있었고, `다음날 07:33`이라는 절대 시각을 보지 못했음.
- 그래서 전날 숙박 종료가 다음날로 넘어간 상황에서도 현재 일정 시작을 당일 07:33으로 비교해 보정시간이 0으로 눌렸음.

### Action
- `getAbsoluteTimelineItemEndMinutes`, `getPreviousMainPlanEntryByIndex`를 추가해 날짜 경계를 포함한 절대 타임라인 기준으로 이전 일정 종료를 계산하도록 수정.
- `getBufferDisplayState`와 `recalculateScheduleAcrossDays`의 고정 시작시간 계산을 `dayIdx * 1440 + HH:MM` 기준으로 바꿔 다음날 첫 일정도 올바른 절대 시각으로 비교되게 보정.

### Result
- 숙박 다음날 첫 일정에 운영 시작 자동 보정을 적용해도 남는 시간이 이동칩 보정시간으로 정상 반영됨.
- 날짜 경계를 지난 뒤에도 고정 시작시간, 보정시간, 이동칩 표시가 서로 같은 기준으로 동작하게 정리됨.

### Bug
- 날짜가 바뀌는 지점에서 다음날 첫 일정이 전날 마지막 일정 종료 시각과 연동되지 않아, 이동시간/보정시간이 실제 흐름과 다르게 끊겨 보이는 문제가 있었음.

### Cause
- `recalculateSchedule`가 Day 단위 배열만 재계산하고 있어, 다음날 첫 일정은 전날 마지막 일정 종료를 모른 채 자기 Day 안에서만 계산되고 있었음.
- 숙박 뒤 Day 분리 정규화는 있었지만, 분리 후 다시 전체 타임라인을 순차 계산하는 단계가 없었음.

### Action
- `recalculateScheduleAcrossDays`를 추가해 `days` 전체를 앞에서 뒤로 순차 재계산하도록 수정.
- 숙박 Day 분리 정규화 이후에도 전날 마지막 일정 종료, 이동시간, 보정시간을 기준으로 다음날 첫 일정이 이어지게 보정.
- `itinerary.days` 변경 시 전체 Day를 다시 순차 계산하고, 이후 `recalculateLodgeDurations`로 숙박 체크아웃/소요시간을 맞추는 후처리를 추가.

### Result
- 날짜 구분은 유지하면서도 시간 계산은 전날 마지막 일정에서 다음날 첫 일정으로 끊김 없이 이어짐.
- 다음날 첫 일정의 시작시각, 이동칩, 보정시간이 전날 숙박/마지막 일정 종료와 일관되게 맞춰짐.

### Bug
- 시작 시간을 임의로 뒤로 미루고 `시작시간 잠금`을 켜도 이동칩 보정시간이 `0분`으로 남는 케이스가 있었음.
- 특히 운영 안내 문구 클릭 보정, 시간 휠 드래그 입력, 잠금 조합에 따라 반영 여부가 달라져 사용자가 일관되게 신뢰하기 어려웠음.

### Cause
- `applyBusinessWarningFix`가 일반 시작시간 수정 경로와 다른 방식으로 보정시간/플래그를 직접 만져서, 이후 재계산 흐름과 상태가 어긋날 수 있었음.
- `setStartTimeValue`가 시작시간 수정 시 Day 전체의 `isDurationFixed`를 강제로 해제하고 있어, 잠금 상태에 따라 시간 흐름이 달라지는 부작용이 있었음.
- 화면은 저장된 `bufferTimeOverride`만 그대로 보여줘서, 고정 시작시간 기준 실제 보정치와 표시값이 어긋날 수 있었음.

### Action
- 고정 시작시간 일정의 보정시간을 `이전 일정 종료 + 이동시간` 대비 실제 고정 시작시각 차이로 다시 계산하는 `getBufferDisplayState`/`syncBufferWithFixedStart` 경로를 정리.
- 운영 안내 문구 클릭도 별도 수동 버퍼 세팅을 제거하고, 일반 `시작시간 고정 + 재계산 + 보정시간 동기화` 흐름으로 통일.
- `setStartTimeValue`에서 Day 전체 `isDurationFixed`를 강제 해제하던 로직 제거.
- 메인 타임라인 이동칩과 네비게이션 보정칩 모두 저장값 대신 실제 고정 시작시간 기준 계산된 보정시간을 표시하도록 수정.

### Result
- 시작시간을 직접 늦추거나 안내 문구로 운영 시작 시각에 맞췄을 때, 이동칩 보정시간이 실제 차이만큼 안정적으로 반영됨.
- 잠금 조합 때문에 보정시간 표시가 `0분`으로 남는 현상이 줄고, 화면 표시와 실제 타임라인 계산 기준이 일치하게 정리됨.

## 2026-03-12

### Bug
- 시간 모달에서 시작/종료를 조절해도 일부 일정이 여전히 소요시간 고정값에 묶여 밀리지 않는 현상이 남아 있었음.

### Cause
- 과거 데이터의 `isDurationFixed`가 남아 있어, 시간 모달에서 시간 변경 후 재계산 시에도 소요시간 잠금 제약이 계속 적용됐음.

### Action
- `setStartTimeValue`, `setPlanEndTimeValue`에서 시간 모달 입력이 들어오면 해당 Day의 일반 일정(backup/ship 제외) `isDurationFixed`를 일괄 해제하도록 보강.

### Result
- 시간 모달 조절 시 레거시 소요시간 잠금의 간섭 없이 뒤 일정 재계산/밀림이 동작하도록 정리됨.

### Bug
- 시간 편집 모달의 드래그 휠이 일부 브라우저/입력 조건에서 거의 반응하지 않거나 조작 중 끊기는 현상이 있었음.

### Cause
- Pointer 이벤트 기반 드래그 구현이 환경별 캡처/이동 이벤트 차이에 민감했고, 내부 스크롤과 포인터 캡처 충돌이 간헐적으로 발생했음.

### Action
- 휠 드래그 입력을 Pointer 중심에서 `mouse/touch` 전역 이벤트 방식으로 교체.
- `mousedown/touchstart`에서 드래그 시작, `window`의 `mousemove/touchmove`로 스크롤 업데이트, `mouseup/touchend`에서 스냅 종료로 통일.
- 드래그 중 모달 닫힘 방지 플래그는 그대로 유지.

### Result
- 시간 모달에서 드래그 입력 반응성이 안정화되어 시작/종료 시간 이동이 확실히 동작함.

### Bug
- 시간 모달에서 종료 시각을 조정해도 뒤 일정 시작시각이 기존 값에 묶여 밀리지 않는 케이스가 있었음.

### Cause
- 뒤 일정들에 남아 있던 `isTimeFixed`가 재계산 시 우선 적용되어, 종료 변경에 따른 연쇄 밀림이 중간에서 고정값에 다시 잡히고 있었음.

### Action
- `setPlanEndTimeValue`에서 종료 시각 직접 조정 시, 뒤 일정을 순회하며 하드 고정(페리/숙박) 전까지 `isTimeFixed`/`isEndTimeFixed`를 해제하도록 보강.
- 이후 `recalculateSchedule`로 재계산해 종료 변경값이 뒤 일정으로 자연스럽게 전달되게 수정.

### Result
- 종료 시각을 모달에서 바꾸면 뒤 일정이 실제로 밀리며, 페리/숙박 같은 고정 장벽 지점에서만 멈추는 흐름으로 정리됨.

### Bug
- 시간 모달에서 시작/종료 조절이 제한 로직에 묶여 실제로 시간을 바꾸기 어려운 상태가 발생했음.
- 불가능 시간대 빨간 표시 기준이 체감상 모호해 오히려 입력 혼란을 유발했음.

### Cause
- 최대 시작시간/최소 종료시간 clamp와 비활성 값 스냅 로직이 동시에 동작하면서, 사용자가 드래그한 입력이 자주 보정되어 조작감이 끊겼음.

### Action
- 시간 모달의 비활성(빨간) 판정과 관련 스냅/보정 로직을 제거.
- 시작/종료 시/분 휠을 모두 24h/60분 전체 범위로 열고, 분 래핑 시만 시 캐리/보로우를 적용.
- 분/시 변경 결과는 하루 분(`0..1439`)으로 정규화해 항상 입력이 반영되도록 수정.

### Result
- 시간 모달에서 시작/종료 시간이 막히지 않고 100% 드래그 입력 가능해짐.
- 빨간 불가 표시 없이, 사용자가 지정한 시간 기준으로 이후 일정 재계산 흐름이 유지됨.

### Bug
- 종료 시간 휠을 드래그할 때 모달이 간헐적으로 닫히는 문제가 있었음.
- 고정 조건 때문에 더 이상 이동할 수 없는 시간 범위가 사용자에게 구분되지 않아 조작이 헷갈렸음.

### Cause
- 휠 드래그와 모달 외부 닫힘 감지가 동시에 반응하는 타이밍이 있었음.
- 휠은 불가능 값을 숨기거나 clamp만 하고 있어, 현재 어떤 값이 막혀 있는지 시각 정보가 부족했음.

### Action
- 휠 드래그 상태를 추적해 드래그 중에는 모달 외부 닫힘 감지를 무시하도록 수정.
- 시간 모달의 스크롤 기반 자동 닫힘을 제거해 드래그 중 오동작을 줄임.
- 휠에 비활성 값 판정(`isValueDisabled`)을 추가해 이동 불가 범위를 빨간색으로 표시.
- 비활성 값으로 스냅되면 가장 가까운 유효 값으로 자동 보정되게 처리.

### Result
- 종료 시간 휠 드래그 중 모달이 꺼지는 현상이 크게 줄었고, 계산 불가 시간대가 빨간색으로 명확히 보여 조작 혼란이 줄어듦.

### Bug
- 시간 편집 모달에서 적용한 선택칩 스타일과 시간셀 카드 스타일이 달라 시각 톤이 분리돼 보였음.

### Cause
- 시간셀은 기존 회색 카드 스타일을 유지하고, 모달만 파란 선택칩 계열로 갱신돼 디자인 언어가 맞지 않았음.

### Action
- 시간셀 시작/종료 카드의 경계, 배경, 숫자 타이포를 모달 선택칩 톤으로 조정.
- 가운데 소요칩도 동일 계열(`border-[#bfd7ff]`, `bg-[#eef5ff]`)로 정리해 카드 전체 스타일 통일.

### Result
- 시간 편집 모달과 시간셀이 같은 디자인 언어로 연결되어 보이고, 시간 정보 인지가 더 일관되게 됨.

### Bug
- 시간 편집 모달에서 분 휠이 `01 -> 00`까지만 내려가고 `59, 58 ...`로 순환되지 않아 시간 보로우(시 -1)가 되지 않았음.

### Cause
- 분 컬럼이 단일 목록의 끝에서 멈추는 비순환 스크롤 구조였고, 분 이동 시 시(hour) 캐리/보로우 로직이 없었음.

### Action
- `TimeWheelColumn`에 순환(`cyclic`) 모드를 추가해 분 컬럼이 끝에서 끊기지 않게 변경.
- 분 변경 시 `delta` 기반으로 시 증감(캐리/보로우)을 계산하도록 시작/종료 시간 로직을 보강.
- 경계값에서 시작은 `0..maxStartMinutes`, 종료는 `startMinutes..1439` 범위로 clamp.

### Result
- 분 휠에서 `01 -> 00 -> 59 -> 58`처럼 자연스럽게 넘어가며, 시(hour)도 함께 자동 증감됨.

### Bug
- 시간 편집 모달에서 종료 시간을 움직이려 해도 소요시간 조절 UI와 상호작용이 겹치면서 종료 시간이 묶인 것처럼 동작했음.

### Cause
- 소요시간 카드가 여전히 직접 증감/잠금 가능한 편집 UI로 남아 있어, 시작/종료 조절 흐름과 동시에 개입하고 있었음.

### Action
- 시간 편집 모달의 소요시간 카드를 읽기 전용 계산값으로 변경.
- 소요시간 증감 버튼과 잠금/유동 토글을 제거하고, `시작/종료 변경 시 자동 계산` 안내만 유지.

### Result
- 모달은 시작/종료 시간 조절에만 집중되고, 소요시간은 계산 결과만 표시되어 종료 시간 조절 충돌이 줄어듦.

### Bug
- 시간 편집 중 충돌 경고 팝업이 작업 흐름을 끊었고, 시작/소요/종료 조절 영역이 세로로 길어 한눈에 보기 어려웠음.
- 아직 안정화되지 않은 메인 지도 기능이 히어로 영역을 계속 차지하고 있었음.

### Cause
- 시간 충돌 감지가 `window.alert`를 직접 호출하고 있었음.
- 시간 편집 모달이 세로 스택 구조라 비교가 느렸고, 시작/종료 휠도 전체 시간을 다 노출하고 있었음.
- 지도 미리보기는 기능 안정화 전인데도 계속 렌더/계산 경로가 살아 있었음.

### Action
- 시간 충돌은 팝업 대신 상태 문구만 남기도록 수정.
- 시간 편집 모달을 `시작 / 소요 / 종료` 가로 3칸 배치로 재구성.
- 시작/종료 휠은 현재 소요시간 기준으로 가능한 범위까지만 드래그되게 제한.
- 메인 지도 카드와 확대 모달 렌더를 비활성화하고, 페이지 로드 후 경로 미리보기 계산도 중단.

### Result
- 시간 편집 흐름이 팝업에 끊기지 않고, 가능한 범위 안에서만 바로 드래그 조절할 수 있게 됨.
- 지도 기능은 화면과 백그라운드 계산에서 일단 빠진 상태로 정리됨.

### Bug
- 시간 편집 모달 상단 안내와 큰 시각 표시가 과해 실제 조작 영역보다 시선이 분산됐고, 데스크톱에서는 마우스로 직접 끌어 시간을 바꾸기 어려웠음.

### Cause
- 시간 모달이 휠 스크롤 중심으로만 설계돼 있었고, 시작/종료 분 컬럼도 5분 단위로 제한돼 있었음.

### Action

### Bug
- 시간 편집 모달에서 천천히 드래그하면 값이 기존 시간으로 되돌아가고, 빠르게만 동작하는 현상이 있었음.

### Cause
- 드래그 중에도 값 동기화 이펙트가 실행되며 스크롤 위치를 현재 값으로 강제 재정렬해 입력이 끊겼음.
- 터치 입력에서 pointer/touch 이벤트가 중복 개입해 드래그 안정성이 떨어졌음.

### Action
- 드래그 중(`pointer/touch active`)에는 동기화 스크롤 보정을 중단.
- 터치 입력 시 pointer 경로를 무시하고 touch 경로만 사용하도록 분리.
- 시간 휠 컬럼 전체 터치/드래그 입력을 유지해 조작 범위를 확대.

### Result
- 시간 모달에서 느린 드래그도 안정적으로 반영되고 값 되돌림 현상이 줄어듦.

### Bug
- 종료 시간 휠을 아래로 드래그할 때 시작 시간보다 작아지는 값을 선택할 수 있었음.

### Cause
- 종료 시간 입력에서 시작 시간 하한 clamp가 없어 분/시 조합에 따라 시작보다 이른 종료가 들어갈 수 있었음.

### Action
- 시간 모달 종료 시간 휠 변경값에 `종료 >= 시작` 하한 clamp를 추가.
- 종료 시간은 시작 시간 아래로 내려가지 않게 모달 입력 단계에서 제한.

### Result
- 종료 시간이 시작 시간보다 작아지지 않도록 드래그 제한이 적용됨.

### Bug
- 분 휠을 드래그할 때 손을 놓기 전에는 시(hour) 캐리/보로우가 즉시 반영되지 않아 조작이 느리게 느껴졌음.

### Cause
- 값 반영이 스냅(정착) 시점 중심이라 드래그 중 연속 입력에서 시간 갱신이 지연됐음.

### Action
- `TimeWheelColumn`에 드래그 중 실시간 반영(`liveOnDrag`) 경로를 추가.
- 분 컬럼(시작/종료)에 `liveOnDrag`를 적용해 드래그 중 분 변화가 즉시 상위 시간 계산으로 전달되게 변경.

### Result
- 분을 드래그하는 동안 시(hour)도 즉시 캐리/보로우되어 손을 떼기 전부터 시간 변화가 바로 보이도록 개선됨.

### Bug
- 시작/종료 시간이 동일할 때 소요시간이 `0분`이 아니라 `1440분`으로 계산되는 문제가 있었음.

### Cause
- 종료 시간 계산에서 `종료 <= 시작`을 다음날로 처리해 동일 시각도 +24h로 넘어갔음.

### Action
- 종료 시간 계산 분기를 `종료 < 시작`일 때만 다음날 보정하도록 수정.

### Result
- 동일 시각 입력 시 소요시간이 `0분`으로 정상 계산됨.

### Bug
- 숙박 계열 종료 시간 휠이 `23시`까지만 보여 심야/다음날 종료를 직접 선택할 수 없었고, 새벽 체크인 후 오전 체크아웃 같은 케이스도 모호하게 처리됐음.

### Cause
- 종료 시간 휠이 항상 `0~23` 시계 한 바퀴만 사용했고, 숙박의 종료값도 단순 `시계값 비교`로만 다음날 여부를 추정했음.

### Action
- 숙박 계열 종료 시간 휠을 `0~47` 절대시각 범위로 확장.
- 표시값은 `08`, `08+`처럼 시계값은 유지하되 `+`로 다음날 구분.
- 숙박 계열 종료 시간 변경은 `setPlanEndAbsoluteMinutes(...)`를 통해 절대 종료 시각 기준으로 반영하도록 수정.

### Result
- 숙박 종료 시간을 당일 오전/다음날 오전으로 구분해 직접 선택할 수 있고, 심야 일정도 `23시 이후` 범위까지 안정적으로 설정 가능해짐.
