# Bug Fix Log

## 2026-03-12

### Bug
- 일정 카드 사이 이동거리 계산이 `주소확인`으로 반복 표시됨.
- 주소 문자열은 정상인데도 전체 경로 재계산 후 실패 캐시가 쌓이면서 실제 좌표가 있는 구간까지 실패처럼 보였음.

### Cause
- 경로 미리보기와 DB 저장은 `geo` 필드를 쓰기 시작했지만, 실제 구간 거리 계산 fallback은 여전히 별도 Nominatim 재조회에 의존했음.
- 한국 주소가 이미 저장돼 있어도 OSRM fallback 직전 좌표 재조회가 실패하면 `주소확인`으로 분류됐음.

### Action
- 일정/내 장소/페리 endpoint에 저장된 `geo`, `geoStart`, `geoEnd`를 구간 거리 계산에서도 우선 사용하도록 수정.
- 저장 좌표가 없거나 주소와 불일치할 때만 `geocodeAddress()` 재조회로 fallback 하도록 변경.
- 동일 계열 버그가 다시 나오면 이 파일에 증상, 원인, 조치사항을 계속 누적 기록.

### Result
- 주소가 이미 저장된 구간은 불필요한 재조회 없이 바로 거리 계산 가능.
- `주소확인`은 실제로 출발지/도착지 좌표를 끝내 못 찾은 경우에 더 가깝게 축소됨.
